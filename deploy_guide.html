Deploy to GitHub Pages

Choose your preferred method and follow the steps

Method 1: GitHub Desktop Easiest

  1. Download GitHub Desktop
    Visit: desktop.github.com
    Install and sign in with: uttamkqr
  2. Create New Repository
    File ‚Üí New Repository
    Name: portfolio
    Description: Professional Portfolio - Data Science & ML
  3. Copy All Files
    From: D:\portfolio\
    To: Repository folder (GitHub Desktop shows location)
  4. Commit Changes
    Summary: "Initial portfolio commit"
    Click: "Commit to main"
  5. Publish Repository
    Click: "Publish repository"
    Make it PUBLIC
    Click: "Publish"
  6. Enable GitHub Pages
    Go to: Settings ‚Üí Pages
    Source: main / (root)
    Save & Wait 2 minutes

Method 2: Git Commands Advanced

  1. Open PowerShell/Terminal
    Navigate to portfolio folder:
    cd D:\portfolio
  2. Initialize Git
    git init
  3. Configure Git (First Time Only)
    git config --global user.name "Uttam Kumar"
    git config --global user.email "uttam16.kqr@gmail.com"
  4. Add All Files
    git add .
  5. Commit Changes
    git commit -m "Initial commit: Professional portfolio"
  6. Create Main Branch
    git branch -M main
  7. Add Remote Repository
    git remote add origin https://github.com/uttamkqr/portfolio.git
    ‚ö†Ô∏è Important: Create repository on GitHub first!
    Go to: github.com/new
    Name: portfolio (Public, no README)
  8. Push to GitHub
    git push -u origin main
    If asked for credentials:
    Username: uttamkqr
    Password: Personal Access Token
    (Not your GitHub password!)
  9. Enable Pages
    Repository Settings ‚Üí Pages
    Source: main / (root) ‚Üí Save

Method 3: Upload via GitHub.com Simplest

  1. Create Repository
    Go to: github.com/new
    Name: portfolio
    Make it: Public
    Click: Create repository
  2. Upload Files
    Click: "uploading an existing file"
    Drag & drop all files from D:\portfolio\
    Or click: "choose your files"
  3. Commit Upload
    Message: "Initial portfolio commit"
    Click: "Commit changes"
  4. Enable Pages
    Settings ‚Üí Pages
    Source: main / (root)
    Save
